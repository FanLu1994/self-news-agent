## 重点推荐

1. JimLiu/baoyu-design：在本地运行 Claude 的 UI 设计能力**
- **来源:** GitHub Trending
- **一句话:** 一个开源项目，让你在本地使用 Cursor、Claude Code 等工具，调用 Claude 生成完全不依赖网页端的精美 UI 原型和演示文稿。

- **链接:** https://github.com/JimLiu/baoyu-design
2. Harness 工程：在智能体优先的世界中运用 Codex**
- **来源:** AI HOT / OpenAI 官方博客
- **一句话:** OpenAI 官方分享的工程实践文章，详细讲述了如何将 Codex 整合进 Harness 的工程流程，以适应“智能体优先”的开发模式。
- **为什么值得关注:** 这是来自第一方的最新实践指引，直接关系到如何让 Codex 成为你团队中的高效“队友”，对于所有追求 AI 辅助编码的开发者都非常有参考价值。
- **链接:** https://openai.com/index/harness-engineering
3. Harness-1：基于强化学习训练的 20B 检索子智能体**
- **来源:** AI HOT / MarkTechPost
- **一句话:** UIUC 与 Chroma 联合推出 Harness-1，一个通过强化学习在有状态搜索框架内训练的、20B 参数的检索子智能体。
- **为什么值得关注:** 它展示了用强化学习 (RL) 专门训练一个“检索”小模型（而非仅依赖 LLM 的检索能力），这样的专业子智能体 (Subagent) 架构在复杂搜索和 RAG 系统中可能有更高的效率和准确性。
- **链接:** https://www.marktechpost.com/2026/06/06/meet-harness-1-a-20b-retrieval-subagent-trained-with-reinforcement-learning-inside-a-stateful-search-harness-on-gpt-oss-20b

## 趋势判断与深度阅读

**趋势判断**
1.  **从“工具”到“代理生态”**：无论是 OpenAI 准备将 ChatGPT 升级为集成 Codex 的平台，还是 Harness 的工程实践，都表明 AI 正从单一问答工具，向能自主执行任务的“智能代理”平台转变。开发者需要关注代理的编排、协作与工程化。
2.  **专业子智能体 (Subagent) 兴起**：像 Harness-1 这样用强化学习专门训练一个“会检索”的智能体，预示着未来 AI 系统不再是单一的“大模型万能”，而是由多个更小、更专业、经过特殊训练的“子智能体”协同工作，以提高特定任务的效率和效果。
3.  **AI 编程能力走出“屏幕”**：北海道农民的案例生动地表明，Codex 和 LLM 的价值已不仅限于生成代码，它们作为“大脑”与 IoT 硬件、API 结合，可以解决物理世界中的复杂问题，这会引发新一轮的“AI + 行业应用”创新。

**深度阅读**
*   **《Harness 工程：在智能体优先的世界中运用 Codex》**
    *   **为什么值得花时间:** 这篇文章来自 OpenAI 官方，不是技术文档，而是回顾工程实践。它描述了在一个大规模工程组织中，如何设计和构建一个允许 Codex 这样的智能体有效运作的“Harness”（工程框架）。如果你正在尝试将 AI 编码助手融入团队工作流，这篇是必读的“官方指南”。
    *   **链接:** https://openai.com/index/harness-engineering
