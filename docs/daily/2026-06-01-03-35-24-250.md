## 今日最值得看

本周 AI 开发者的焦点集中在两个方向：**Rust 重写的本地工具链**和**Apple 端侧模型的落地路径**。前者以毫秒级 Office 文档处理 MCP 服务器为代表，预示着 AI Agent 从“能跑”向“高效、廉价”迈进的趋势；后者则展示了巨头如何用蒸馏技术将大模型塞进手机，这对于关注边缘计算和隐私计算的开发者极具参考价值。同时，GitHub 上的 Token 付费争议也提醒我们，AI 应用的经济模型正成为新的博弈点。

## 重点推荐

1. **毫秒级 Office 文档 MCP 服务器：opendocswork-mcp**
*   **来源**: GitHub Trending
*   **一句话**: 一个用 Rust 编写的、能本地处理 Excel、Word、PPT 文件的 MCP 服务器。
*   **为什么值得看**: 对于构建 AI Agent 的开发者来说，文档处理是刚需。这个项目用 Rust 原生实现了亚毫秒级的性能，并且完全本地优先、开源。这意味着你可以用它为自己的 Agent 配上一个极其高效、低延迟的“Office 助手”，同时不用担心数据外泄。它解决了一个具体的、高频的痛点。
*   **链接**: https://github.com/Aimino-Tech/opendocswork-mcp
2. **工程化 RAG 文档助手：DocPaws**
*   **来源**: GitHub Trending
*   **一句话**: 一个集成了知识库、PDF 索引、Agent 工具编排和引用溯源功能的 RAG 工程化项目。
*   **为什么值得看**: 它不仅仅是一个 RAG demo。项目实现了 “scope 检索” 和 “拒答阈值” 等企业级需求，这通常是生产环境中 RAG 系统的关键难点。对于想搭建可靠文档问答系统的开发者，这个 FastAPI + Vue3 的开源项目是一个很好的起点或参考模版。
*   **风险提示**: 目前 stars 数不多 (57)，社区成熟度和长期维护性有待观察。
*   **链接**: https://github.com/biao994/DocPaws
3. **AI 友好的 PPT 生成工具链：GordenPPTSkill**
*   **来源**: GitHub Trending
*   **一句话**: 一套包含 17 套精美模板和 Python 脚本，通过 `edits.json` 配置即可生成 PPT 的自动化方案。
*   **为什么值得看**: 它巧妙地将 AI 不擅长的“排版设计”和擅长的“内容生成”分离开。开发者可以用 AI 生成 `edits.json`，然后利用这个工具生成专业的 PPTX 文件。这是一个实用的 AI + 办公自动化范例，对于需要批量生成报告或演示文稿的场景非常有价值。
*   **链接**: https://github.com/GordenSun/GordenPPTSkill
4. **苹果 WWDC 将推本地 AI：从 Gemini 蒸馏的小模型**
*   **来源**: AI HOT / X
*   **一句话**: 爆料称苹果将在 WWDC 上展示运行在 iPhone 上的、从 Google Gemini 蒸馏而来的 Siri 和 AI 模型。

## 趋势判断与深度阅读

**趋势判断**
1.  **Rust 正在成为 AI 工具链的“新基建”**：从 `opendocswork-mcp` 可以看出，追求极致的性能和本地化正成为 AI 工具链的重要方向。Rust 凭借其零成本抽象和安全性，开始从基础设施下沉到面向开发者的具体工具中，特别是 MCP 这类代理交互协议。
