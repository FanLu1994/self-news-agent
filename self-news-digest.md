
# 简介
基于现有的self-news-agent开发。  
实现每日自动获取 ve2x、hacker news、github trend、 linuxdo 等论坛的人们帖子，进行归纳总结，每天更新两次，自动生成一份md文档，并且把最新的填充到readme，并且更新后可以把内容发送到指定邮箱。

自动统计话题分类，并且在readme添加话题历史趋势。
# 需求详情

## 1. 数据源

| 来源 | 内容 | 频率 |
|------|------|------|
| Ve2x | 技术讨论 | 每日 |
| Hacker News | 科技资讯 | 每日 |
| GitHub Trending | 热门项目 | 每日 |
| Linux.do | 社区讨论 | 每日 |
| Reddit (r/programming, r/MachineLearning) | 技术讨论 | 每日 |
| Product Hunt | 产品发现 | 每日 |

## 2. 运行机制

- **更新频率**: 每日 2 次（建议时间：9:00、18:00）
- **运行方式**: 自动定时任务
- **处理逻辑**: 抓取 → 去重 → 归纳总结 → 生成文档

## 3. 输出格式

- **文档格式**: Markdown
- **存储位置**: `docs/daily/YYYY-MM-DD.md`
- **README 更新**: 自动同步最新摘要到项目 README.md

## 4. 邮件推送

- **触发时机**: 文档生成后
- **目标**: 发送到指定邮箱
- **内容**: 摘要正文 + 链接到完整文档

## 5. 总结风格

- 简洁扼要，突出重点
- 按来源分类展示
- 包含标题、链接、核心观点

## 6. 话题分类与趋势

- **自动分类**: AI 归纳每条内容的所属话题（如 AI、前端、后端、DevOps、创业等）
- **话题统计**: 统计每日各话题的帖子数量
- **趋势展示**: README 中展示话题历史趋势图（最近 7/30 天的热门话题变化）
- **存储格式**: JSON 文件存储每日统计数据，便于生成趋势图表
